Matchmaker: A Generic Framework for Consumer-Producer Matching

Overview

This project aims to create a generic framework to match consumers of a product with its producers, à la Uber or AirBnB. The reason such a framework might be desirable is that it could free producers from the hegemony of the network provider, by allowing then to easily bring up their own matching service.

It is a hypothesis of the project that this family of services have enough in common that a generic framework can cover a large enough portion of what goes on that creating a specific service can be done (mostly?) by non-programmers using templates.

Server API

End points

  • get_agent(agent_id)
    agent_id may be email address.
    This loads and returns agent's persistent characteristics.
  • get_matches(agent, details (as part of agent?))
    Load producer agents into env. Match consumer agent's current prefs to best producers.
    Returns list of matching agents.
  • book(agent, other, details)
    Make a booking.
    Returns Success or Error (with explanation).
  • cancel(booking_id)
    Cancel a booking.
    Returns Success or Error (with explanation).
